## 概要

GitHub Copilotを企業に導入する際のセキュリティおよび運用設計上の課題を整理します。  
これを通して、GitHub Copilotを安全かつ効果的に導入できるよう支援することを目指します。

## セキュリティ

GitHub Copilotを企業に導入する際には、いくつかセキュリティ面で調査・認識しておくべきことがあります。  
ここでは、導入にあたって特に重要な4つの観点について整理します。

### 通信経路が安全か調査する

まず確認すべきは、GitHub Copilotの通信経路が安全かどうかです。  
通信経路が安全ではないなら、通信時にソースコードなどの流出リスクがあるということです。

Copilotは、エディタとGitHubのサーバー間でのデータのやり取りを暗号化されたHTTPSプロトコルで行います。  
これにより、**通信内容の漏洩リスクが低減**されます。

> Copilot 関連のデータは、転送時に Transport Layer Security (TLS) を使用して暗号化され  
> -- [GitHub Copilot Trust Center - GitHub Resources](https://resources.github.com/ja/copilot-trust-center/)

そもそもどのようなデータが送受信されるのかについては以下のようにまとめられています。

> 処理される個人データの種類には次のものが含まれます。  
> - ユーザーエンゲージメントデータ: これには、承認または却下された完了、エラーメッセージ、システムログ、製品使用状況メトリクスなど、Copilot とのユーザー対話で取得された仮名化されたID が含まれます。
> - プロンプト: これらは、コンテキストとともにチャットまたはコードの入力であり、提案を生成するために Copilot の AI に送信されます。
> - 提案: これらは、プロンプトに基づいてユーザーに提供される AI 生成のコード行またはチャット応答です。
> - フィードバック データ: これは、サポート チケットからのフィードバックに加えて、反応 (サムアップ/ダウンなど) やオプションのコメントを含むリアルタイムのユーザー フィードバックで構成されます。
>
> -- [GitHub Copilot Trust Center - GitHub Resources](https://resources.github.com/ja/copilot-trust-center/)

### 保存されるデータが何か調査する

次に確認すべきは、GitHub Copilotの保存するデータが何かです。  
仮に保存されるなら、保存された場所からの流出リスクがあるということです。

重要なのはプロンプトと提案でしょう。  
これらはIDE経由かつ、Copilot BusinessとCopilot Enterpriseにおいては**一切保存されない**ことが明記されています。

> （筆者注）Copilot BusinessとCopilot Enterpriseについての記載が以下  
> チャットとコード補完のために IDE 経由でアクセスします。
> - プロンプトと提案: 保持されません
> - ユーザーエンゲージメントデータ: 2 年間保存されます。
> - フィードバック データ: 本来の目的に必要な期間保存されます。
>
> 他のすべての GitHub Copilot は以下にアクセスして使用します。
> - プロンプトと提案: 28 日間保持されます。
> - ユーザーエンゲージメントデータ: 2 年間保存されます。
> - フィードバック データ: 本来の目的に必要な期間保存されます。
>
> -- [GitHub Copilot Trust Center - GitHub Resources](https://resources.github.com/ja/copilot-trust-center/)

### モデルのトレーニングに利用されるか調査する

次に確認すべきは、ユーザのデータがAIモデルのトレーニングに利用されるかどうかです。  
仮にトレーニングに使用されるなら、他のCopilot利用者にソースコードなどが流出するリスクがあるということです。

Copilot BusinessとCopilot Enterpriseにおいては**モデルのトレーニングに利用されない**ことが明記されています。  
逆に言えば、Copilot Individualはそうではないということが読み取れます。

> （筆者注）GitHub は、GitHub のモデルをトレーニングするために Copilot Business または Enterprise データを使用しますか?という質問に対する回答が以下  
> いいえ。GitHub はモデルのトレーニングに Copilot Business データも Enterprise データも使用しません。  
> -- [GitHub Copilot Trust Center - GitHub Resources](https://resources.github.com/ja/copilot-trust-center/)

### 著作権およびライセンス違反リスクがあるか調査する

最後に、著作権およびライセンス違反のリスクがあるかどうかです。

これはけっこう微妙なところです。  
あるかないかで言えばないとは言えないものの、**発生確率は低い**のではというのが筆者の肌感です。

詳しいことは[GitHub Copilot Trust Center - GitHub Resources](https://resources.github.com/ja/copilot-trust-center/)の「GitHub Copilot と著作権」というセクションにまとめられているのでご覧ください。

## 運用設計

運用設計上の課題を整理します。  
とはいえ、このあたりは企業ごとにやるべきことが大きく変わるところだと思います。  
したがって「具体的にこうすべき！」ということは言わずに「こういうことを運用設計で考えるといいかもよ」という肌感でまとめます。

### 最適な利用プランを選ぶ

[GitHub](https://github.co.jp/pricing.html)、[GitHub Copilot](https://docs.github.com/ja/copilot/about-github-copilot/subscription-plans-for-github-copilot)それぞれの利用プランを検討します。

基本的な考え方は、GitHubのアカウントに対してGitHub Copilotのシート（利用権限のようなもの）を紐づけるというイメージです。

注意すべきことは、個人で無料アカウントは1つのみということです。  
私用の無料アカウントと会社用の無料アカウントを作ることは規約違反です。

> 1 人の個人または 1 法人が維持できる無料アカウントは 1 つのみです  
> -- [GitHub のサービス使用条件 - GitHub Docs](https://docs.github.com/ja/site-policy/github-terms/github-terms-of-service#3-account-requirements)

### ガイドライン・ルールを作る

利用時のガイドライン・ルールを作ります。

たとえば、以下のような事項を明文化します。  
利用者視点、運用者視点それぞれで洗い出すのがコツです。

- 利用者
  - ときどきアンケート取るから協力してね
  - 機密情報や個人情報は入力しないでね
  - AIの出力は鵜呑みにしないで裏取りとかきちんと取ってね
- 運用責任者
  - 毎月利用者数と費用をきちんと報告してね
  - 定期的に効果検証を実施してCopilotの継続利用判断を行ってね

### 利用開始時・終了時のフローを確立する

窓口はどこで、どんな申請を行い、シートの割当手順がこれで...などフローを確立しましょう。  
また、忘れがちですが利用を辞める際の流れも定義しておきましょう。

### 効果測定の方法を決める

重要かつ難しい部分です。  
以下のような情報を使ってCopilotの有用性を示します。

- 管理画面からわかるコストや利用者数など
- アンケートなどを使った定性評価
- Four Keysなどを使った定量評価

ここでの成果物は、試験導入など本導入の前段階でも使えます。

### コストの管理方法を明文化する

コストを見る画面は用意されているので、どこにアクセスすればいいのか、前月のコストを見るにはどうすればいいのか、今月のコスト予測はどうなのか、といった疑問に答えられる手順書などを用意しておきましょう。

どんなタイミングでどんな情報が必要なのかは、マネージャーや費用管理部門などと連携を取りつつ、その人たちが求める情報をすぐに出せるようにしましょう。

### 利用促進をもたらすツール概要資料を作る

今まで紹介したことを頑張って調査・ドキュメントにまとめたところで利用してもらわないと意味がありません。  
GitHub Copilotとはどんなもので、何が嬉しくて、どうやって使い始めればいいのか、このあたりの情報を簡潔に得られる資料があると便利です。

コードをあんまり書かない偉い人が見て「有用そうだ」と思ってもらえるくらいが理想です。

## まとめ

GitHub Copilotを安全かつ効果的に導入するために以下を調査・検討しましょう。

- セキュリティ
  - 通信経路が安全か調査する
  - 保存されるデータが何か調査する
  - モデルのトレーニングに利用されるか調査する
  - 著作権およびライセンス違反リスクがあるか調査する
- 運用設計
  - 最適な利用プランを選ぶ
  - ガイドライン・ルールを作る
  - 利用開始時・終了時のフローを確立する
  - 効果測定の方法を決める
  - コストの管理方法を明文化する
  - 利用促進をもたらすツール概要資料を作る
